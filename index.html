Builder의 Template은 ReactJS로 구성되어 있으며, Webpack을 통해서 Spring 실행 시, View Template 영역으로 통합하여 사용 Redis Server를 이용하여 별도의 Session으로 여러 Project의 로그인 정보를 한곳에 모아서 관리 Redis Server의 Cache Server기능을 이용하여 대용량 검색 데이터를 관리 [Ex - Medical Coding] MongoDB의 시스템 구조를 이용하여, XML로 변환이 필요한 동적 데이터를 JSON으로 관리 실제 환경에서는 MS Azure를 이용하여 Cloud Server를 운용하였으며, Azure Load Balancer를 사용
Spring Data JPA를 통해 Class 단위로 Entity 관리 및 JPA Auditing 처리 [데이터 등록/수정 시 기록 관리] Spring Data Envers를 통해 데이터베이스 특정 테이블 자동 기록 관리 위 시스템의 Entity에 대한 데이터베이스를 common, builder, cdms로 분류하여 Transaction Manager를 나누어 관리 Swagger를 통해서 외부 시스템과 인터페이스 구축 시, Restful API등을 정의하는 데에 사용 / 관련 API 문서 제공 iText 라이브러리를 통해, 증례기록서(CRF)에 대한 PDF 문서를 생성하는데에 사용 및 htmlToPdf 모듈로 html기반으로 PDF를 생성 Docker Container를 사용하여 로컬 테스트 진행 시, 가상의 MS-SQL / NginX / Redis Server를 생성하여 사용 Jakarta mail API를 이용하여 SMTP 이메일 전송 서비스 구성 [초대, 알림 등] 해당 시스템은 비영리 임상표준기관 CDISC의 ODM-XML이라는 임상 규격을 준수하여 시스템을 설계하였으며, JAXB 라이브러리를 통해, ODM-XML에 대한 클래스를 정의, XML <-> Java Class 간 데이터 변환을 수행하여 시스템 내 데이터를 운용.
SweetAlert2 라이브러리를 통해 Smooth한 Modal화면 구현 Bootbox.js 라이브러리를 통해 다양한 종류의 Dialog 화면 구현 blockUI 플러그인을 통해 Data Loading 화면 구현 React DOM Router를 통해 React환경 내 URL 관리 Redux / Redux-Saga로, 새로고침 또는 페이지 로딩 시 세션에 등록된 유저 정보 및 Metadata에 대한 데이터 호출에 사용 Promise Tracker를 통해, Redux-Saga에서 수행하는 비동기 API 호출에 의한 응답 동작을 보조 Zustand를 사용하여 공용 데이터에 대한 상태 관리를 수행 React Query를 활용하여 공용 데이터에 대한 호출, 처리 등을 수행 Ant Design을 활용하여 View에 전체적으로 활용되는 Form 또는 레이아웃 디자인 등을 구성 React Beautiful dnd 라이브러리를 통해 eCRF 생성 시 Drag & Drop 구조의 직관적인 설정 방식을 구축 React 18next 라이브러리를 통해 다국어 환경 설정, react-cookies를 활용하여 백엔드 내에 설정되어있는 언어 설정과 동기화 처리 C3.js/D3.js 라이브러리를 통해 CDMS 환경의 Dashboard Chart 표현 DevExtreme 컴포넌트를 사용하여, Data Grid (Table) 및 통계자료 생성 등에 사용
